# Carta do Projeto

## Visite Caçapava - Plataforma de Turismo Gamificada

---

## 1. Informações do Projeto

| Campo | Valor |
|-------|-------|
| **Nome do Projeto** | Visite Caçapava |
| **Desenvolvedor** | Oryum Tech - Software House 360° |
| **Cliente** | Secretaria de Turismo de Caçapava do Sul |
| **Data de Início** | Janeiro 2025 |
| **Data de Conclusão MVP** | Novembro 2025 |

---

## 2. Visão do Projeto

Criar uma plataforma digital inovadora que promova o turismo em Caçapava do Sul através de gamificação, proporcionando uma experiência interativa e envolvente para visitantes enquanto gera dados valiosos para a gestão pública do turismo.

---

## 3. Objetivos

### Objetivos de Negócio

1. Aumentar o fluxo turístico em Caçapava do Sul
2. Melhorar a experiência do visitante
3. Coletar dados para planejamento turístico
4. Promover pontos turísticos menos conhecidos

### Objetivos Técnicos

1. Desenvolver MVP funcional em 10 meses
2. Implementar sistema de gamificação robusto
3. Integrar inteligência artificial para assistência
4. Garantir performance e usabilidade

---

## 4. Escopo

### Incluído

- ✅ Sistema de gamificação (pontos, badges, leaderboard)
- ✅ Check-in em pontos turísticos via QR Code
- ✅ Validação por GPS
- ✅ Dashboard administrativo
- ✅ Sistema de hotéis e restaurantes
- ✅ Assistente virtual com IA
- ✅ Mapas interativos
- ✅ Rotas temáticas

### Não Incluído (MVP)

- ❌ Aplicativo mobile nativo
- ❌ Sistema de pagamentos
- ❌ Reservas online
- ❌ Backend dedicado

---

## 5. Stakeholders

| Stakeholder | Papel | Interesse |
|-------------|-------|-----------|
| Secretaria de Turismo | Cliente | Gestão do turismo |
| Turistas | Usuários finais | Experiência de viagem |

| Restaurantes | Parceiros | Feedback de clientes |
| Oryum Tech | Desenvolvedor | Entrega do projeto |

---

## 6. Premissas

1. Acesso à internet disponível nos pontos turísticos
2. Turistas possuem smartphones com GPS
3. Dados de POIs fornecidos pela Secretaria
4. APIs de IA disponíveis durante o projeto

---

## 7. Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|---------------|---------|-----------|
| Atraso no desenvolvimento | Média | Alto | Metodologia ágil |
| Problemas com APIs de IA | Baixa | Médio | Sistema de fallback |
| Baixa adoção | Média | Alto | Marketing e incentivos |
| Fraude em check-ins | Média | Médio | Validação criptográfica |
